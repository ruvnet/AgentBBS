## 1. Summary [STABLE]

`late-web` is the public Axum web frontend for `late.sh`.

It serves Askama-rendered pages and static assets for:

- Landing (`/`, `/status`) and audio pairing (`/{token}`, `/pair/status`)
- Browser TUI demo/tunnel view (`/play`)
- Read-only Artboard gallery (`/gallery`)
- Public work profiles (`/profiles`)
- Internal/demo dashboard (`/dashboard`)
- Stable audio stream proxy (`/stream`)

The crate depends on `late-core` for DB access, telemetry, shutdown handling, and shared models. It depends on `dartboard-core` for decoding Artboard snapshot canvases.

---

## 2. Entrypoints And App Shape [STABLE]

- `src/main.rs` is the binary entrypoint. It initializes `late_core` telemetry with service name `late-web`, loads `Config::from_env()`, builds a no-proxy `reqwest::Client`, creates `late_core::db::Db::from_env()`, binds `0.0.0.0:LATE_WEB_PORT`, and shuts down through `late_core::shutdown::wait_for_shutdown_signal()`.
- `src/lib.rs` exposes `AppState { config, db, http_client }` and `app(state) -> Router`.
- `app(state)` merges `pages::router()`, serves `/static` from `late-web/static`, adds `/test`, redirects unknown routes to `/`, wraps the router with `late_core::telemetry::http_telemetry_middleware`, then attaches state.
- `/test` intentionally returns an `AppError` for exercising error/telemetry behavior.

### Runtime Config [STABLE]

Required environment variables:

- `LATE_WEB_PORT`: web HTTP listen port.
- `LATE_SSH_INTERNAL_URL`: server-side URL for `late-ssh` API calls, currently `/api/now-playing`.
- `LATE_SSH_PUBLIC_URL`: browser-reachable `late-ssh` API host/base URL for WebSocket routes.
- `LATE_AUDIO_URL`: upstream Icecast/audio base URL proxied by `/stream`.
- `LATE_WEB_TUNNEL_TOKEN`: shared token used by `/play` when connecting to `late-ssh` `/api/ws/tunnel`; must be non-empty.

DB config is delegated to `late_core::db::Db::from_env()`.

Local defaults are generated by the root `Makefile`; production values live in `infra/service-web.tf`.

## 3. Routes [STABLE]

`src/pages/mod.rs` is the route composition root.

| Route | Owner | Purpose |
|---|---|---|
| `GET /` | `pages/home` | Landing page with SSH/CLI copy actions and links into gallery/play/profiles. |
| `GET /status` | `pages/home` | HTMX now-playing/listeners fragment for the landing footer. |
| `GET /{token}` | `pages/connect` | Audio pairing page for an SSH session token. |
| `GET /pair/status` | `pages/connect` | HTMX status fragment for the pairing track display. |
| `GET /dashboard` | `pages/dashboard` | Internal/demo dashboard page. |
| `GET /dashboard/now-playing` | `pages/dashboard` | HTMX now-playing partial. |
| `GET /dashboard/status` | `pages/dashboard` | HTMX pseudo system-status partial. |
| `GET /gallery?key=...` | `pages/gallery` | Read-only Artboard snapshot gallery from DB. |
| `GET /play` | `pages/play` | Full-screen xterm.js browser TUI demo through web tunnel. |
| `GET /play/listeners` | `pages/play` | HTMX listener-count fragment. |
| `GET /profiles` | `pages/profiles` | Public work profile index. |
| `GET /profiles/{slug}` | `pages/profiles` | Public work profile detail, or 404 template. |
| `GET /stream` | `pages/stream` | Long-running `audio/mpeg` proxy to Icecast with silence fallback. |
| `GET /test` | `lib.rs` | Error simulation endpoint. |
| fallback | `lib.rs` | Temporary redirect to `/`. |

Important route details:

- `connect` audio pairing sends paired-client `client_state`, `viz`, and fallback `heartbeat` payloads to `late-ssh` `/api/ws/pair`.
- `play` builds `/api/ws/tunnel?token=...` from `LATE_SSH_PUBLIC_URL` and `LATE_WEB_TUNNEL_TOKEN`, converting HTTPS to WSS, HTTP/local to WS, and bare public hosts to WSS.
- `gallery` reads saved `artboard_snapshots` rows through `late_core::models::artboard::Snapshot`; it does not read live `late-ssh` memory. Default selection falls back to `main`, then first daily, then first monthly snapshot.
- `profiles` reads `WorkProfile`, `Profile`, `Showcase`, and `User` models from DB. The index limits to 100 and sorts `open`, then `casual`, then `not-looking`. Detail pages render profile bio through sanitized Markdown with raw HTML stripped, split comma-separated contact entries, and only create links for `http://`, `https://`, or email-like values.
- `stream` normalizes `LATE_AUDIO_URL` so `/stream` is appended once.

## 4. Templates And Static Assets [STABLE]

- `askama.toml` sets the Askama template root to `src`.
- `src/pages/app.html` is the global HTML shell. It loads `/static/style.css`, HTMX from unpkg, and Alpine from jsDelivr. Landing/social metadata frames `late.sh` as a terminal clubhouse for terminal people, not developers-only.
- `src/pages/layout.html` extends `app.html` with the centered default page layout.
- Full-screen experiences such as `/play` and `/gallery` extend `app.html` directly.
- HTMX partial templates live beside their page owner, e.g. `dashboard/partial_now_playing.html`, `dashboard/partial_status.html`, `home/status.html`, and `connect/status.html`.
- Tailwind source is `static/input.css`; compiled output is committed as `static/style.css`.
- npm scripts in `late-web/package.json`:
  - `npm run tailwind:watch`
  - `npm run tailwind:build`
- Static fonts include Symbols Nerd Font under `static/fonts/`; `/play` relies on these for terminal glyph rendering.

## 5. Integration Contracts [STABLE]

### late-ssh API And WebSockets

- Server-side now-playing fetches call `{LATE_SSH_INTERNAL_URL}/api/now-playing` from `pages/shared/now_playing.rs`.
- Browser audio pairing connects to `ws(s)://{LATE_SSH_PUBLIC_URL}/api/ws/pair?token={token}`.
- Browser TUI demo connects to `ws(s)://{LATE_SSH_PUBLIC_URL}/api/ws/tunnel?token={LATE_WEB_TUNNEL_TOKEN}` with `cols` and `rows` query params.

### Audio Pairing

Browser audio pairing owns the web implementation of the paired-client protocol:

- `client_state`: `{ event: "client_state", client_kind: "browser", muted, volume_percent }`
- `viz`: `{ event: "viz", position_ms, bands, rms }`
- `heartbeat`: fallback when Web Audio analysis is unavailable

The browser uses Web Audio `AnalyserNode`; the CLI uses a Rust FFT. They share schema, not identical numbers.

### Stream Proxy

`/stream` must:

- Return `200 OK`.
- Set `Content-Type: audio/mpeg`.
- Set no-store/no-cache headers.
- Keep yielding audio bytes indefinitely.
- Yield non-empty bundled silence chunks from `assets/silence.mp3` when upstream is down, errors, or ends.
- Retry the upstream every 500ms after a silence chunk.

This endpoint is used by browser pairing and by local CLI workflows.

### Database-Backed Pages

- `/gallery` decodes `Snapshot.canvas` as `dartboard_core::Canvas` and `Snapshot.provenance` as `cells: Vec<(Pos, String)>`.
- `/profiles` uses public work-profile slugs and user/profile/showcase data.
- These pages require the DB pool in `AppState`; route tests that exercise them should use DB-backed integration infrastructure.

## 6. Error, Telemetry, Metrics [STABLE]

- `src/error.rs` maps `AppError::Internal` and `AppError::Render` to HTTP 500.
- The error page uses `pages/error.html`; if rendering the error page fails, the response falls back to plain text.
- Current error responses include internal error text in the rendered body. Be careful before attaching secrets or sensitive upstream context to errors.
- Router-level HTTP telemetry comes from `late_core::telemetry::http_telemetry_middleware`.
- Outbound now-playing requests use `send_traced()` so trace headers propagate to `late-ssh`.
- `metrics.rs` is no-op unless built with the `otel` feature.
- Current web metrics:
  - `late_web_page_views_total{page,has_token}`
  - `late_web_now_playing_fetch_total{result}`
- Preserve non-`otel` compilation when adding metrics.

## 7. Testing And Verification [STABLE]

Repo policy from `../CONTEXT.md` applies:

- LLM agents must not run `cargo test`, `cargo nextest`, or `cargo clippy` in this repo.
- Human owner runs verification manually.
- Expected full gate before merge:

```bash
cargo fmt --all -- --check
cargo clippy --workspace --all-targets -- -D warnings
cargo nextest run --workspace --all-targets
```

`late-web` test placement:

- Handler/route behavior belongs in `late-web/tests/*`.
- Pure page/model transformations belong in inline `#[cfg(test)]` modules under `src/pages/*`.
- Error mapping tests belong in `src/error.rs`.
- DB-touching integration tests should use the shared testcontainers path from `late-core::test_utils`.

Current tests:

- `tests/dashboard.rs`: spawns the web app and a mock now-playing server; asserts dashboard and HTMX partial rendering.
- `tests/stream.rs`: verifies `/stream` upstream passthrough and silence fallback.
- Inline pure tests cover status value generation, stream URL/silence helpers, gallery labels/titles, profile formatting helpers, tunnel URL conversion, and error mapping.

Known web smoke-test exception:

- `tests/dashboard.rs` and `tests/stream.rs` currently create an inert `Db::new(&DbConfig::default())` because they do not touch DB routes. This is only acceptable for non-DB route smoke tests. If a web integration test uses DB-backed routes (`/gallery`, `/profiles`), use `late_core::test_utils::test_db()` instead of the inert DB shortcut.

## 8. High-Risk Invariants [STABLE]

- `LATE_SSH_PUBLIC_URL` must be browser-reachable. Server-internal service names are valid for `LATE_SSH_INTERNAL_URL`, not for browser WS URLs.
- `play::tunnel_ws_url()` scheme handling is user-visible: HTTPS -> WSS, HTTP/local -> WS, bare public host -> WSS.
- Web Audio `createMediaElementSource()` is one-shot per `<audio>` element. Create the `AudioContext` and source node once, then suspend/resume; do not close and recreate on every play.
- Pairing WS/audio retries are bounded. A WS drop should not kill audio playback UI while the audio stream is still working.
- `connect/page.html` sends analyzer frames every 100ms while playing; keep payload shape compatible with `late-ssh` paired-client handling.
- Gallery JSON is embedded into a page script context. Keep the explicit `</` to `<\/` replacement or use an equivalent safe embedding strategy.
- Gallery rendering depends on `dartboard-core` canvas/provenance schema. Coordinate with `late-ssh/src/app/artboard/CONTEXT.md` and `late-core` migrations before changing it.
- Profile status strings are presentation-coupled to `open`, `casual`, and `not-looking` for ordering, labels, and CSS state.
- The `/stream` fallback must never yield empty chunks; some clients treat empty reads as stream end.
- `metrics.rs` must compile both with and without the optional `otel` feature.

## 9. Local Development [VOLATILE]

Full Docker dev stack:

```bash
make start
```

Native Rust services with Docker infrastructure:

```bash
docker compose up -d postgres icecast liquidsoap
cargo run -p late-ssh
cargo run -p late-web
```

Tailwind rebuild:

```bash
cd late-web
npm run tailwind:build
```

Do not assume `cargo run -p late-web` has enough env by itself. It needs the required `LATE_WEB_*`, `LATE_SSH_*`, `LATE_AUDIO_URL`, and DB env vars from `.env` or the shell.
